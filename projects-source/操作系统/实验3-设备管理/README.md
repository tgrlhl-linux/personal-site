## 程序架构

### 核心类

#### 1. ProcessRequest 类
表示一个等待访问磁盘的进程请求

**属性：**
- `process_name`: 进程名称，格式为 P1, P2, P3...
- `cylinder`: 进程要求访问的磁道号

#### 2. DiskScheduler 类
磁盘调度器主类，实现四种调度算法

**属性：**
- `cylinders`: 磁盘柱面总数，默认为200
- `start_cylinder`: 初始磁头位置，默认为100
- `requests`: 存储所有请求的磁道号列表
- `request_table`: 存储所有请求的ProcessRequest对象列表

### 主要方法

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `generate_requests(length)` | 生成随机进程请求I/O表 | request_table |
| `print_request_table()` | 打印格式化的进程请求I/O表 | None |
| `fcfs()` | 先来先服务算法 | (sequence, total, avg) |
| `sstf()` | 最短寻道时间优先算法 | (sequence, total, avg) |
| `look()` | 电梯调度算法 | (sequence, total, avg) |
| `clook()` | 单向扫描算法 | (sequence, total, avg) |
| `run_all()` | 运行所有算法并输出结果 | results |

## 程序功能

### 1. 初始化进程请求I/O表
程序会随机生成指定长度的进程请求序列：

```
进程请求 I/O 表
========================================
|   进程名   |   要求访问的磁道号   |
----------------------------------------
|    P1     |         55          |
|    P2     |         90          |
|    P3     |         35          |
========================================
```

### 2. 四种调度算法

#### FCFS (先来先服务)
- **原理**: 按照进程请求的先后顺序进行调度，不考虑磁道位置
- **优点**: 简单公平，每个请求都能得到处理
- **缺点**: 平均寻道时间长
- **适用场景**: 进程较少时

#### SSTF (最短寻道时间优先)
- **原理**: 优先选择距离当前磁头最近的请求，使每次寻道时间最短
- **优点**: 平均寻道时间较短
- **缺点**: 可能导致饥饿现象
- **适用场景**: 随机访问模式

#### LOOK (电梯调度)
- **原理**: 磁头沿一个方向移动，处理完该方向所有请求后反向移动
- **优点**: 性能较好，避免饥饿
- **缺点**: 实现较复杂
- **适用场景**: 一般场景

#### CLOOK (单向扫描)
- **原理**: 对LOOK算法的改进，磁头只沿一个方向移动，到达最远端后直接回到另一端
- **优点**: 响应时间均匀
- **缺点**: 有额外移动距离
- **适用场景**: 实时系统

### 3. 输出格式
每个算法显示详细的执行过程：

### 4. 可视化选项

程序提供四种可视化模式：


#### 静态可视化

- 2x2子图布局，便于直观对比

#### 动态可视化
- 实时演示磁头移动过程
- 红色圆点表示当前磁头位置
- 绿色圆点表示已访问的柱面
- 支持按Enter继续，输入'q'跳过剩余动画

## 算法特性对比

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| FCFS | 简单公平 | 平均寻道时间长 | 进程较少时 |
| SSTF | 寻道时间短 | 可能导致饥饿 | 随机访问模式 |
| LOOK | 性能较好，无饥饿 | 实现较复杂 | 一般场景 |
| CLOOK | 响应时间均匀 | 有额外移动 | 实时系统 |

## 实验结果分析要点

运行程序后，可以从以下角度分析：
1. **平均寻道长度**: SSTF 和 LOOK 通常表现较好
2. **公平性**: FCFS 最公平，但性能最差
3. **饥饿现象**: SSTF 可能导致饥饿，LOOK 和 CLOOK 可以避免
4. **响应时间**: CLOOK 的响应时间更均匀

## 程序使用示例

```python
# 创建调度器
scheduler = DiskScheduler(cylinders=200, start_cylinder=100)
 
# 生成请求序列
scheduler.generate_requests(length=10)
 
# 运行单个算法
sequence, total, avg = scheduler.fcfs()
sequence, total, avg = scheduler.sstf()
sequence, total, avg = scheduler.look()
sequence, total, avg = scheduler.clook()

# 运行所有算法（带可视化）
results = scheduler.run_all(length=10, show_static=True, show_animation=False)
```
