# 进程调度模拟实验

## 项目概述

基于动态优先级的时间片轮转调度算法模拟器。C++ 实现调度核心逻辑，HTML/JS 实现可视化。支持四种经典调度策略的综合模拟：先来先服务、短作业优先、优先级调度和时间片轮转，通过一个统一的动态优先级框架实现。

## 架构设计

程序分为两个独立模块：

调度引擎（scheduler.cpp）：核心基于 Scheduler 类，维护进程池和优先队列就绪队列。每次调度循环中，将到达进程移入就绪队列，选择队首进程运行一个时间片，若进程未完成则重新计算优先级并放回队列。四种调度策略的差异体现在优先级计算方式和队列排序逻辑上。

可视化前端（visualization.html）：纯前端实现，解析 C++ 程序输出的文本日志，还原就绪队列、运行队列、完成队列的状态变化，并实时生成甘特图。step() 函数逐事件推进，play() 实现自动播放，所有状态由前端独立维护。

## 技术亮点

动态优先级调整：就绪队列中的进程每等待一个时间片优先级递增，正在运行的进程执行一个时间片后根据公式 newPriority = base - remainingTime + waitingTime/2 重新计算。剩余时间越短的进程优先级越高，等待越久的进程获得补偿，兼顾了短作业优先和防饥饿。

时间片轮转与优先级抢占的融合：时间片到期触发优先级重算，同时就绪队列中等待的进程优先级递增，实现了轮转调度的同时保留了优先级调度的区分度。

可视化与引擎解耦：调度器只输出格式化文本，可视化层通过正则解析逐行还原状态，两者无需 API 耦合。视觉采用赛博朋克风格，支持逐帧步进和自动播放。

## 设计决策

选择 priority_queue 实现就绪队列而非手动维护有序链表，利用 STL 二叉堆在插入和取顶操作上 O(log n) 的复杂度，足够满足教学模拟场景。每次操作就绪队列时需遍历更新等待时间，这一 O(n) 操作在进程数控制在 10 个以内时影响可忽略。

## 关键代码解读

```cpp
void adjustPriority(PCB& pcb) {
    int waitBonus = pcb.waitingTime / 2;
    int newPriority = priorityBase - pcb.remainingTime + waitBonus;
    if (newPriority < 1) newPriority = 1;
    pcb.priority = newPriority;
}
```

该函数是调度策略的核心：剩余时间越短优先级越高（鼓励短作业），等待时间越长优先级补偿越多（防止饥饿）。waitBonus 使优先级成为动态值，系统能自适应调整调度倾向。visualization.html 中的 step() 函数按事件类型分别处理到达、开始、完成和量子到期四种状态，将调度日志可视化还原。
