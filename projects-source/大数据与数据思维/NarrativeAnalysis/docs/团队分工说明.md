# 团队分工说明

**项目：** 叙事计算 —— 基于电影剧本大数据的叙事结构挖掘与分析系统
**课程：** 大数据与数据思维 · 期末大作业

---

## 分工总表

| 成员 | 角色 | 工作量 | 具体任务 |
|------|------|--------|----------|
| **童国睿** | 项目 Leader / 数据管道 | 35% | 数据采集、预处理 Pipeline、HBase 建表、Flask API、文档答辩PPT |
| **杨超越** | MR 核心开发 / 集群运维 | 35% | MR Job 编写、Docker 集群搭建、YARN 调优、JAR 打包 |
| **何锦辉** | 前端可视化 | 25% | Vue 3 + ECharts + D3.js 前端开发、数据展示 |
| **李景颖** | 算法设计 / 性能优化 | 30% | ECR 算法、叙事指纹、K-Means 聚类、情感词典扩充 |

*※ 工作量占比有交叉。算法同学参与部分 MR 代码（侧重 Job4/5），全栈同学配合做特征提取。*

## 数据流向

```
童国睿: 下载 → 清洗 → 数据管道 → HDFS/HBase
杨超越: HDFS → MR Job1(情感) + MR Job2(共现) → 结果文件
李景颖: 词典 + 分析提炼 ← 结果解读
何锦辉: MR 结果 → ECharts 前端展示
```

*注：各人负责模块之间有交叉配合，上述分工对应设计方案第八章，是项目初始规划。*
