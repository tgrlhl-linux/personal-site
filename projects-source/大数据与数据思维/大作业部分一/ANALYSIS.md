# HDFS 数据处理 Pipeline

> **与本课程完整项目的关系**：本目录是「[大数据叙事分析系统](/projects/narrative-analysis)」的数据管道组件（作业一）。完整的叙事分析系统在此基础上增加了 Flask API 后端和 Web 可视化前端。

## 项目概述

大数据课程作业的数据预处理与导入管道，负责将电影剧本数据从原始格式清洗、转换并导入 Hadoop HBase。Pipeline 支持三种运行模式：完整流程（full）、仅生成 API 数据（api）、仅写入 HBase（hbase），可根据集群状态灵活选择。

## 架构设计

Pipeline 按数据流向分为三个阶段：

情感词典加载：从 emotion_dict.csv 加载情感词典，每词包含 positive 和 negative 两个维度的情感权重。词典作为剧本分析的参考基准，是整个计算过程的基础。

剧本分析引擎：analyze_script() 函数解析剧本文本，识别场景标记 [SCENE] 和全大写角色名进行场景分割和角色提取。对无场景标记的剧本，按 50 行窗口自动分割。每个场景计算情感总分（正面词计数减去负面词计数后除以总词数归一化），同时统计角色共现关系。分析结果输出为结构化的 JSON 数据。

HBase 导入：通过 happybase 客户端连接 HBase Thrift 接口，将分析结果写入三张表：movie_emotion（场景情感向量）、character_network（角色共现网络）、narrative_pattern（叙事模式特征）。

## 技术亮点

自适应场景分割：同时支持两种场景格式——有 [SCENE] 标记的结构化剧本和无标记的纯文本剧本。对于后者，采用正则 `[A-Z][A-Z\s\.]{1,30}` 识别角色名并按固定窗口分割场景，保证了不同格式剧本的统一处理。

三表 HBase 存储设计：情感数据、共现数据、叙事模式分别存入独立的 HBase 表，列族设计对应不同的查询模式。emotion 列族适合范围扫描场景情感序列，cooccur 列族支持高效的角色对查询，features 列族存储叙事指纹用于聚类分析。

角色共现网络：通过对场景中角色同时出现的次数做统计，生成 Top-30 的角色共现关系。这些数据可进一步用于构建社交网络图，分析剧本中角色互动强度。

## 设计决策

Pipeline 采用 argparse 实现三种运行模式，使数据处理可以分阶段执行。在没有 Hadoop 集群的环境中可以只跑 API 模式生成 JSON 数据供前端使用，HBase 写入作为可选步骤，保证了开发环境的兼容性。

## 关键代码解读

```python
net_emotion = round((pos_count - neg_count) / max(total_words, 1), 6)
```

情感计算公式将原始的情感词计数值归一化为 [-1, 1] 区间的净情感值，消除了场景长度对情感强度的影响。这种归一化使得不同长度的剧本/场景之间可以公平比较情感变化趋势。
