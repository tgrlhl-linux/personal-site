## ② 预处理 Pipeline + HDFS 导入

**设计方案对应：** 第三章 3.3 预处理流程（Step 1~6）

### 交付内容

| 文件 | 说明 |
|------|------|
| `scripts/preprocess.py` | 剧本清洗脚本：场景识别 + 角色提取 + 文本清洗 |
| `scripts/upload_to_hdfs.sh` | HDFS 数据上传脚本 |
| `scripts/data_pipeline.py` | 全链路数据管道：预处理 → 情感分析 → JSON/HBase 输出 |

### 实现细节

**Step 1 - 场景分割：** 正则匹配 INT./EXT./SCENE 标记，用 `[SCENE]` 标注边界。对无标准标记的剧本（Memento、Shawshank 等），自动降级为固定窗口分析（窗口大小 50 行）。

**Step 2 - 角色提取：** 识别全大写角色名（ALL-CAPS 模式 `^[A-Z][A-Z\s\.]{1,30}$`），建立角色-场景映射矩阵，为共现分析提供基础。

**Step 3 - 文本清洗：** 去除 HTML 标签、`&nbsp;` 实体、版权声明行、网址、多余空行，统一 UTF-8 编码。

**Step 4~6 - 数据上传：** `upload_to_hdfs.sh` 自动将清洗后的数据 cp 到 Docker master 容器，执行 `hadoop fs -put` 导入 HDFS。

### 清洗结果

| 总场景数 | 1,879 个 |

## ③ HBase 建表与数据写入

**设计方案对应：** 第五章 5.2 HBase 表设计

### 交付内容

| 文件 | 说明 |
|------|------|
| `scripts/hbase_schema.sh` | HBase 表创建脚本（3 张表） |
| `scripts/data_pipeline.py` | 数据写入模块（含 HBase Thrift 写入 + 文件 fallback） |

### 表结构设计

#### 表 1: `movie_emotion` — 场景情感向量

| 属性 | 值 |
|------|-----|
| RowKey | `movie_id` (String) |
| CF:emotion | `scene_XXXX` → JSON 情感向量 |
| CF:meta | `title`, `scenes`, `characters` |

#### 表 2: `character_network` — 角色共现矩阵

| 属性 | 值 |
|------|-----|
| RowKey | `movie_id` (String) |
| CF:cooccur | `charA#charB` → 共现次数 |
| CF:role | 角色出场信息 |

#### 表 3: `narrative_pattern` — 叙事模式特征

| 属性 | 值 |
|------|-----|
| RowKey | `movie_id` (String) |
| CF:features | `fingerprint` — 15 维叙事特征向量 |
| CF:acts | `count`, `boundaries` |
| CF:cluster | 聚类标签 |

### 数据管道架构

```
原始剧本 → preprocess.py → 清洗文本
                               ↓
                      data_pipeline.py
                         ↓        ↓
                    JSON文件    HBase 表
                    (API用)   (3张表)
```

## ④ 后端 API 服务开发

**设计方案对应：** 第二章 2.1 技术栈选型（后端 API: Flask）/ 第六、6.2 交互设计

### 交付内容

| 文件 | 说明 |
|------|------|
| `api/app.py` | Flask REST API 服务（5 个端点 + 自动数据加载） |
| `api/requirements.txt` | Python 依赖清单 |
| `data/api_data/` | 预生成的结构化 JSON 数据集 |

### API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/` | GET | API 首页，列出所有可用端点 |
| `/movies` | GET | 获取所有电影列表（含基本信息） |
| `/movies/<id>` | GET | 获取单部电影完整分析数据 |
| `/movies/<id>/emotion` | GET | 获取情感曲线（含 ECR 转折点检测） |
| `/movies/<id>/cooccurrence` | GET | 获取角色共现数据 |
| `/stats` | GET | 数据集统计信息 |

### 启动方式

```bash
pip install flask flask-cors
python api/app.py
# 服务运行在 http://localhost:5000
```

### 情感变化率（ECR）算法实现

在 `/movies/<id>/emotion` 端点中实现了**简化版 ECR 场景边界检测**（对应设计方案第四章 4.4）：

$$ECR(t) = |e_t - e_{t-1}| + |e_{t+1} - e_t|$$

当 ECR > 0.05 时标记为候选转折点，返回 Top-10。答辩时可直接调用 API 展示。
