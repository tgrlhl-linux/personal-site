# Shell 成绩管理系统

## 项目概述

基于 Bash 的全功能成绩管理系统，实现学生成绩的增删改查、排序统计、三级权限控制和多维度报表导出。数据存储于 CSV 文件，通过 Shell 脚本完成所有操作，是一次将 Shell 编程能力推向工程化边界的尝试。

## 架构设计

系统采用函数式模块化设计，主流程为：数据初始化 -> 班级配置加载 -> 用户认证 -> 主菜单循环。核心数据流通过 get_data_lines() 函数实现用户数据隔离，admin 看到全部数据，teacher 看到管辖班级，student 看到本人记录。

功能模块分为四个层级：基础 CRUD（添加/删除/修改/查询学生）、排序（按科目/总分/平均分）、统计报表（分布/平均分/GPA/挂科名单）、加分模块（班级维度统计/学院汇总/用户管理/报表导出）。每个子功能对应独立的函数，通过子菜单组织调用。

## 技术亮点

三级权限与数据隔离：用户系统支持 admin/teacher/student 三种角色，通过 SESSION_ROLE 和 SESSION_SCOPE 变量控制数据可见性和操作权限。teacher 只能操作管辖班级的学生，student 只能查询自己，admin 拥有全部权限。get_data_lines() 函数根据角色动态过滤数据行。

纯 Shell 的数值计算方案：由于 Bash 不支持浮点运算，系统封装了 awk_add/awk_mul/awk_div 等工具函数，所有数值计算委托给 awk 完成。grade_to_num() 函数将优秀/良好/中等/及格/不及格等中文等级映射为数值分数，支持混合评分体系。

批量录入功能：支持一次输入多行"学号 成绩"数据批量写入 CSV，学号支持短号自动补全（后 4 位自动加前缀 20181828），教师视角自动验证管辖班级权限。这是期末登分场景下最实用的功能。

## 设计决策

选择 CSV 文件存储而非数据库，因为作业要求纯 Shell 实现，且 CSV 的文本格式便于 sed/awk 直接操作。class_info.txt 作为班级-学院-系的映射配置文件，比硬编码更灵活。报表导出到独立的 report/ 目录，每次导出带时间戳防止覆盖。

## 关键代码解读

```bash
get_data_lines() {
    case "$SESSION_ROLE" in
        admin)    echo "$all_data" ;;
        teacher)  # 只返回管辖班级的学生行 ;;
        student)  grep "^$SESSION_SCOPE," ;;
    esac
}
```

该函数是整个权限系统的枢纽。所有查询、排序、统计操作都通过 get_data_lines 获取数据，而非直接读取 CSV 文件，确保数据隔离在源头实现而非分散在各函数中。这种"一次过滤、多处使用"的设计在 Shell 脚本中尤为简洁高效。
